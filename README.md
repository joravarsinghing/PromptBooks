# PromptBooks

**PromptBooks** is an Android proof-of-concept app for AI-assisted bookkeeping. The core idea is simple: a user enters a transaction in natural language, the app uses AI to interpret it, saves the transaction locally, and reflects the data in a dashboard. 

---

## Quick overview

- **Purpose:** Demo-quality, chat-first bookkeeping POC that shows AI interpretation → local storage → dashboard reflection.
- **Scope:** Proof‑of‑concept only — not a production accounting system. Focus is on demo credibility, clear UX, and a believable foundation for future work. 

---

## Tech stack

- **Language:** Kotlin  
- **UI:** XML layouts + Fragments  
- **Architecture:** Simple Fragment-driven (not full MVVM)  
- **Networking:** Retrofit  
- **AI Provider:** OpenRouter API  
- **Database:** Room (local)  
- **Build:** Gradle Kotlin DSL  
- **Theme:** AppCompat light  
- **Main package:** `com.example.promptbooks`

**Pinned versions (MVP):** Gradle 8.10.2; AGP 8.6.1; Kotlin 1.9.24; Room 2.6.1.

---

## What works (MVP features)

- Chat-based natural-language transaction entry.
- App-side guardrail to block vague inputs before calling AI.
- AI interprets transaction intent and fields; app saves structured transactions to Room.
- Confirmation replies generated by the app.
- Dashboard with **Bank Balance**, **Sales**, and **Recent Transactions**.
- Delete per-row via 3-dot menu; **Clear Data** for demo reset.
- **New Chat** clears visible chat only.
- **Generate Sample Data** from support popup.
- Placeholder popups for attachments and mic.
- Chat session persists when switching tabs.
- Date/time shown on dashboard rows.
- Signed APK generation tested.

---

## Deliberately not implemented (for clarity)

- Attachment parsing or OCR
- Real voice input
- Reports or PDF export
- Bank sync or Zoho Books integration
- Multi-user accounts or production accounting rules
- Full transaction edit/detail popup (Milestone 2 addresses this)

---

## AI design principles

- **AI = interpreter only.** The app owns database writes, dashboard calculations, confirmation wording, guardrails, and display formatting.  
- **Do not** rely on AI to compute dashboard totals or authoritative values. The app must compute and persist those values.

---

## UX & visual guidelines

- **Chat-first**: chat is the primary interaction surface; it should feel like a lightweight bookkeeping assistant.  
- **Guardrail message** for vague inputs: `I need a bit more detail. Please include the amount and whether it was a purchase, sale, expense, or income.`
- **Color system** (approved palette):
    - PromptBooks Blue: `#3484A9`  
    - White: `#FFFFFF`  
    - Background Off-white: `#F8F9FA`  
    - Light Grey Surface: `#F1F3F4`  
    - Divider Grey: `#DADCE0`  
    - Secondary Grey: `#5F6368`  
    - Primary Text Black: `#202124`  
    - Positive Green: `#2E7D32`  
    - Delete/Error Red: `#C62828`  

Rules: use PromptBooks Blue for primary actions; red only for destructive/negative values; green only for positive values.

---

## API key handling

- **Do not** hardcode the OpenRouter API key in Kotlin files.
- Recommended: put `OPENROUTER_API_KEY` in `local.properties` and expose via `BuildConfig.OPENROUTER_API_KEY`. Retrofit should send `Authorization: Bearer ${BuildConfig.OPENROUTER_API_KEY}`.
- **Do not commit** real keys.

---

## Build & device notes

- Lock Gradle/AGP/Kotlin versions for MVP stability; avoid blind AGP upgrades.
- Edge-to-edge UI is intentionally **not** implemented for this MVP; keep content below status bar and above gesture nav.
- Status bar: PromptBooks Blue with white icons.
- Use Android Studio Image Asset workflow for launcher icons; keep splash simple.
